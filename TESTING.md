# Kabegame Plasma Wallpaper Plugin - 测试指南

## 快速测试

### 1. 安装插件

```bash
cd src-plugin
./install.sh
```

### 2. 重启 Plasma Shell

```bash
kquitapp5 plasmashell && kstart5 plasmashell
```

或者直接重新登录。

### 3. 配置插件

1. 右键点击桌面 → **配置桌面和壁纸**
2. 在 **壁纸类型** 下拉菜单中选择 **Kabegame Wallpaper**
3. 点击 **配置** 按钮

## 测试项目

### 基础功能测试

#### 1. 单张壁纸设置

- [ ] 点击 "Browse..." 选择一张图片
- [ ] 确认图片正确显示
- [ ] 尝试不同的填充模式：
  - [ ] Fill (填充)
  - [ ] Fit (适应)
  - [ ] Stretch (拉伸)
  - [ ] Center (居中)
  - [ ] Tile (平铺)

#### 2. 过渡效果测试

准备 2-3 张测试图片，依次设置并观察过渡效果：

- [ ] **None** (无过渡)
  - 应该立即切换，无动画
  
- [ ] **Fade** (淡入淡出)
  - 旧图片应该淡出
  - 新图片应该淡入
  - 过渡应该流畅
  
- [ ] **Slide** (滑动)
  - 新图片应该从右侧滑入
  - 过渡应该流畅
  
- [ ] **Zoom** (缩放)
  - 新图片应该从小到大缩放
  - 同时伴随淡入效果

#### 3. 过渡时长测试

- [ ] 设置不同的过渡时长（100ms, 500ms, 1000ms, 2000ms）
- [ ] 确认过渡动画时长正确

### 轮播功能测试

#### 1. 文件夹轮播

- [ ] 选择一个包含多张图片的文件夹
- [ ] 启用轮播
- [ ] 设置轮播间隔（例如 10 秒）
- [ ] 观察是否自动切换

#### 2. 轮播顺序

- [ ] **Random** (随机)
  - 确认图片以随机顺序播放
  
- [ ] **Sequential** (顺序)
  - 确认图片按顺序播放

### 配置持久化测试

- [ ] 配置壁纸设置
- [ ] 重启 Plasma Shell
- [ ] 确认配置保持不变
- [ ] 确认壁纸正确显示

### 性能测试

- [ ] 切换大尺寸图片（4K, 8K）
  - 观察是否流畅
  - 检查内存使用
  
- [ ] 快速切换多张图片
  - 观察是否有卡顿
  - 检查 CPU 使用

### 边界情况测试

- [ ] 不设置任何图片（空配置）
- [ ] 设置不存在的图片路径
- [ ] 设置无效的文件（非图片）
- [ ] 设置权限不足的文件
- [ ] 在不同分辨率下测试
- [ ] 在多显示器环境下测试

## 调试方法

### 查看日志

```bash
# 实时查看 Plasma Shell 日志
journalctl -f | grep plasmashell

# 或者在终端运行 Plasma Shell
kquitapp5 plasmashell
plasmashell 2>&1 | grep -i "kabegame"
```

### 检查配置文件

```bash
# 查看配置
cat ~/.config/plasma-org.kde.plasma.desktop-appletsrc | grep -A 20 "kabegame"
```

### 手动测试 QML

```bash
# 使用 qmlscene 测试 QML 文件
qmlscene ~/.local/share/plasma/wallpapers/org.kabegame.wallpaper/contents/ui/main.qml
```

## 常见问题

### 问题 1: 插件不出现在壁纸类型列表中

**解决方法**:
1. 确认插件已正确安装到 `~/.local/share/plasma/wallpapers/`
2. 检查 `metadata.json` 文件是否正确
3. 重启 Plasma Shell 或重新登录

### 问题 2: 配置界面不显示

**解决方法**:
1. 检查 `config.qml` 文件是否存在
2. 检查 `main.xml` 配置定义是否正确
3. 查看 Plasma Shell 日志中的错误信息

### 问题 3: 图片不显示

**解决方法**:
1. 确认图片路径正确
2. 确认图片格式支持（PNG, JPG, JPEG, BMP, GIF, WEBP）
3. 检查文件权限
4. 查看控制台日志中的错误

### 问题 4: 过渡效果不生效

**解决方法**:
1. 确认过渡效果已选择（不是 "None"）
2. 确认过渡时长大于 0
3. 尝试切换到不同的图片
4. 检查 QML 动画代码

### 问题 5: 轮播不工作

**解决方法**:
1. 确认轮播已启用
2. 确认文件夹路径正确
3. 确认文件夹包含图片文件
4. 检查 Timer 是否正常运行（查看日志）

## 性能基准

正常情况下的性能指标：

- **内存使用**: < 100MB
- **CPU 使用** (闲置): < 1%
- **CPU 使用** (过渡中): < 10%
- **过渡延迟**: < 50ms
- **图片加载时间** (1080p): < 500ms
- **图片加载时间** (4K): < 1s

如果超出这些指标，可能需要优化。
